Cleaner bootable NITDroid kernel

  • By Chuan Ji
  • |
  • Feb 16, 2011
  • |
  • Tags: Projects
  • |
  • Comments

Alright, based on my previous research, I’ve managed to build a cleaner bootable kernel on top of the Android OMAP kernel 2.6.32, revision 69a7364c16c5aeb246dbb4c4e7696e39841492f5. The diff file is found here. The changes I made are:

  • arch/arm/configs/n810_defconfig: The (current) temporary default configuration for the N810.

  • arch/arm/mach-omap2/{Kconfig,Makefile}: Added switchs and targets providing LCD and USB support on the N8X0.

  • arch/arm/mach-omap2/board-n8x0{,lcd,usb,mmc}.c: Added functional support for the N8X0 board and the corresponding hardware components of the N8X0.

  • arch/arm/plat-omap/include/plat/io.h and arch/arm/mach-omap2/io.c: Backported an API change that the board-n8x0* files depend on and is apparently in upstream linux-omap 2.6.32.

  • arch/arm/mach-omap2/mailbox.c: Fixed compilation warning.

  • arch/arm/plat-omap/include/plat/serial.h and arch/arm/mach-omap2/serial.c: I don’t really understand what’s going on here, but it compiles. This is a direct copy from Luke-Jr’s 2.6.33.

  • arch/arm/plat-omap/{Kconfig,Makefile}: Added switchs and targets for bootloader information passing and the reading of boot reasons and component versions. The kernel will not boot without all three enabled.

  • arch/arm/include/asm/setup.h: Added declarations needed for bootloader information passing.

  • arch/arm/plat-omap/{bootreason.c,common.c,component-version.c}: Provide support for bootloader information passing and the reading of boot reasons and component versions.

  • arch/arm/plat-omap/i2c.c: Fixed stupid compilation error.

  • arch/arm/plat-omap/include/plat/cbus.h: Provides a data structure required by board-n8x0*.c.

  • drivers/input/touchscreen/{Kconfig,Makefile}: Added support for TSC2005, the touchscreen used in the N810.

  • include/linux/spi/tsc2005.h and drivers/input/touchscreen/tsc2005.c: Provide support for TSC2005.

  • drivers/serial/Makefile,drivers/serial/omap-serial.c,arch/arm/plat-omap/include/plat/omap-serial.h: omap-serial.{h,c} are unused and cause compilation errors; removed.

  • drivers/usb/gadget/composite.c: Fixed stupid compilation error.

  • drivers/video/omap/{dispc.c,omapfb.h,rfbi.c}: The changes to these three files are magical. I have no clue how they work, but without them framebuffer doesn’t work and with them it does. It’s like magic. Copied from Luke-Jr’s 2.6.33.

  • drivers/watchdog/omap_wdt.c: I don’t understand how this works either, but it won’t normally compile. Direct copy from Luke-Jr’s 2.6.33.

So what’s next? Well, this is still a really primitive kernel:

  • No audio support

  • No MMC support (I had to disable the Menelaus driver; see previous post)

  • No WiFi support

I will now start working on these.
